					-Introducción

	OpenApi es una especificación de swagger que es un framework para generar apis
	y documentacion.
	Gobernada por la fundacion linux.
	Open source.
	
   ·Historia
	En 2018, William Chen (contribuyente de Swagger Codegen)
	y más de 40 colaboradores bifurcaron todo el código
	hacia un proyecto llamado OpenAPI Generator
   ·Para que sirve
	OpenAPI Generator es una herramienta para generar  APIS RESTFUL,
	documentación, configuración  y tests a partir de una especificación
	OpenAPI

					-Flujo de trabajo

1. especificacion de modelos de la base de datos mediante fichero yaml
2. Comandos para elegir configuracion del proyecto

3.
Mientras haya modelos en yaml:
	 se generan endpoint --> creacion modelo, bd, test y documentacion
Si no hay modelos:
	termina la generación del proyecto


notas:

endpoint: rutas que pones en el navegador y te devuelven datos
modelos: clases con atributos
test: pruebas sobre la bd y los modelos
documentacion: markdowns (archivos de marca para visualizarlo en el navegador)

-				 	CASOS DE USO
Se suelen usar diagramas UML de casos de uso para su representación.

Se suelen usar en la etapa de toma de requisitos pero son de gran ayuda urante el diseño arquitectónico para entender el uso del sistema y quien lo usará.

En este diagrama lo que cabe destacar es:
· La generación de API es la principal funcionalidad del proyecto.
Puede generar una API por defecto pero también se puede modificar un archivo de configuración y así crear automática una API según las especificaciones del propio usuario.
Además, al generar la API, se crea también la documentación de esta.
Además de generar APIs, OpenAPI Generator le permite al usuario la creación automatica de Test una vez generada la API deseada.
Y por último, algo que también podriamos destacar es que OpenAPI Generator existe tambien como una imagen en Docker, que podría ser muy útil en ciertos casos, como empresas grandes ya que con una sola imagen puedes usar OpenAPI.
